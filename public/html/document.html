Project REST API Documentation

Endpoints

Users

  • GET /api/users - Retrieve all users

    Header:

                  Content-Type: application/json

    Example Response:

                    {
                  "id": 101,
                  "username": "matti",
                  "password": "$2b$10$rdUzxAcXahkFKPha89jMLedWGE1mYVlERZchxRG98ak69lB.IhWkG",
                  "role": "user",
                  "address": "Test Address",
                  "created_at": "2025-04-22T13:27:51.000Z",
                  "email": "matti.matti@Testi.Testi",
                  "filename": "file"
                }
  • GET /api/users/:id - Retrieve a user by ID

    Header:

                  Content-Type: application/json

    Example Response:

                    {
                  "id": 101,
                  "username": "matti",
                  "password": "$2b$10$rdUzxAcXahkFKPha89jMLedWGE1mYVlERZchxRG98ak69lB.IhWkG",
                  "role": "user",
                  "address": "Test Address",
                  "created_at": "2025-04-22T13:27:51.000Z",
                  "email": "matti.matti@Testi.Testi",
                  "filename": "file"
                }
  • POST /api/users - Create a new user

    Header:

                  Content-Type: application/json

    Body:

                    {
                  "username": "hfsff",
                  "email": "accchssfg@Testi.Testi",
                  "password": "salasana",
                  "address": "Test Address",
                  "filename": "file"
                }

    Example Response:

                    {
                  "message": "User created successfully",
                  "user_id": 103
                }
  • PUT /api/users/:id - Update a user by ID

    Header:

                  Content-Type: application/json

    Body:

                    {
                  "username": "Testi",
                  "email": "updated.email@test.com",
                  "password": "newpassword",
                  "role": "user",
                  "address": "Updated Address"
                }

    Example Response:

                    {
                  "message": "User updated successfully"
                }
  • DELETE /api/users/:id - Delete a user by ID

    Header:

                  Content-Type: application/json

    Example Response:

                    {
                  "message": "User deleted successfully"
                }

Authentication

  • POST /api/auth/login - Login a user

    Header:

                  Content-Type: application/json

    Body:

                    {
                  "username": "testuser",
                  "password": "password123"
                }

    Example Response:

                    {
                  "user": {
                    "id": 109,
                    "username": "vvvvvv",
                    "password": "$2b$10$0SlzYQZR4e3kONcO0aZBmu0AG4I7HLVF0QyyxrxPtaxAT/AfCt4Ta",
                    "role": "user",
                    "address": "Testitie",
                    "created_at": "2025-04-22T14:20:30.000Z",
                    "email": "vvvvvv@example.com",
                    "filename": "file"
                  },
                  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMDksInVzZXJuYW1lIjoidnZ2dnZ2Iiwicm9sZSI6InVzZXIiLCJpYXQiOjE3NDUzMzE3NTUsImV4cCI6MTc0NTMzNTM1NX0.VuRsp-fcywfGxTBaxFvy0a-WWMgKElY2wTT21swzmVA"
                }
  • POST /api/auth/register - Register a user to the server with a role (requires admin token)

    Header:

                    Content-Type: application/json
                Authorization: Bearer "ADMIN_TOKEN"

    Body:

                    {
                "username": "testuser",
                "email": "testuser@example.com",
                "password": "password123",
                "filename": "file",
                "address": "Testitie",
                "role": "admin"
                }

    Example Response:

                    {
                  "message": "User registered successfully"
                }
  • GET /api/auth/me - Get current user (requires token)

    Header:

                    Authorization: Bearer "TOKEN"
                Content-Type: application/json

    Example Response:

                    {
                  "user_id": 18,
                  "username": "testuser",
                  "iat": 1745329830,
                  "exp": 1745333430
                }
  • GET /api/auth/logout - Logout a user

    Example Response:

                    {
                  "message": "Logged out successfully"
                }
